6. Testing Methodology

6.1 Test Environment

Backend Server Node.js Express (localhost:5000)
AI Model Google Gemini 2.0 Flash
API Endpoint /api/mba-fit/resume/analyze-text
Fallback Method Keyword-based classification with tiebreaker

6.2 Resume Analysis Process

  1. Resume text is extracted from uploaded file
  2. Text is sent to Gemini AI with specialization context
  3. AI analyzes keywords, skills, experience patterns
  4. Fallback keyword matching if AI fails
  5. Results sorted by score with package-based tiebreaker

7. Conclusion & Recommendations

The MBA Fit Finder AI Resume Analyzer demonstrates strong performance with an 86.7% accuracy rate across all 6 MBA specializations. The model performs excellently on clear-cut professional profiles but shows some confusion on cross-domain resumes.

7.1 Key Findings

  • Strengths: 100% accuracy on Finance, Analytics, Marketing, and Operations
  • Weaknesses: Cross-domain profiles (HR Analytics, Startup Finance) get misclassified
  • Pattern: Data science keywords overpower domain-specific context

7.2 Recommendations for Improvement

  • Add context-aware keyword weighting (e.g., "HR Analytics" should boost HR, not Analytics)
  • Implement secondary match presentation when confidence is low
  • Expand test dataset to 50+ resumes with more edge cases
  • Add compound keyword detection for cross-domain terms
